# Figure Management Workflow

Systematic approach to organizing, documenting, and embedding figures in research documents.

## Prerequisites

- Research document (Markdown or LaTeX)
- Figure files (PNG, PDF, SVG recommended)
- Python 3.7+ (for automation scripts)

## Steps

### 1. Organize Figure Directory

Create a structured figure directory:

```bash
research/figures/
├── scoliosis_phase_diagram.png
├── cosserat_rod_schematic.svg
├── energy_landscape.pdf
├── control_stability_plot.png
└── run_20260214_200928_b063c7/   # Simulation outputs
    ├── trajectory.png
    └── convergence.png
```

**Naming conventions:**

- Use descriptive names: `phase_diagram.png` not `fig1.png`
- Use underscores, not spaces: `growth_curve.png`
- Include key info: `scoliosis_phase_diagram_annotated.png`
- Version if needed: `model_v1.png`, `model_v2.png`

### 2. Document Figure Metadata

For each important figure, document:

- **Description**: What does it show?
- **Source**: Generated by which script/analysis?
- **Parameters**: What simulation/analysis parameters?
- **Purpose**: Which document/section uses it?

Create `figures/README.md`:

```markdown
# Figure Documentation

## scoliosis_phase_diagram.png
- **Description**: Phase diagram showing gravity-dominated, cooperative, and information-dominated regimes
- **Generated by**: `scripts/generate_phase_diagram.py`
- **Parameters**: K values [0.1-10], τ values [10-100ms]
- **Used in**: Main theoretical framework, section 3.4

## cosserat_rod_schematic.svg
- **Description**: Schematic of Cosserat rod with curvature vectors
- **Source**: Created manually in Inkscape
- **Used in**: Methods section
```

### 3. Check Existing Figure References

// turbo

```bash
python .agent/skills/research-workflow/scripts/embed_figures.py research/scoliosis_theoretical_framework.md --figures-dir research/figures --verbose
```

This will:

- List all figure references in document
- Check if files exist
- Report missing figures
- Show available figures in directory

### 4. Add Missing Figures

If figures are referenced but files are missing:

**Option A: Generate the figure**

```bash
python scripts/generate_your_figure.py --output research/figures/new_figure.png
```

**Option B: Use placeholder**

```bash
# Create placeholder
echo "Placeholder for missing figure" > research/figures/placeholder.txt
```

### 5. Embed New Figures

**Manual embedding:**

```markdown
![Phase diagram showing three regimes of spinal stability](../figures/scoliosis_phase_diagram.png)
```

**Automated embedding:**

```bash
python .agent/skills/research-workflow/scripts/embed_figures.py research/document.md \
    --add "Phase diagram description" "research/figures/phase_diagram.png" \
    --section "Results"
```

### 6. Validate Figure Quality

Check each figure for:

- **Resolution**: Minimum 300 DPI for publications
- **Readability**: Text visible at target size
- **File Format**:
  - PNG: Photos, complex images
  - PDF/SVG: Diagrams, plots (vector graphics)
  - Avoid JPEG for scientific figures (lossy compression)

### 7. Optimize Figure Files

**Reduce file size:**

```bash
# PNG compression
pngquant --quality=80-95 input.png --output output.png

# PDF optimization  
gs -sDEVICE=pdfwrite -dCompatibilityLevel=1.4 -dPDFSETTINGS=/ebook input.pdf output.pdf
```

**Convert formats:**

```bash
# SVG to PDF (for LaTeX)
inkscape figure.svg --export-pdf=figure.pdf

# PNG to PDF
convert figure.png figure.pdf
```

### 8. Create Figure Captions

Good captions include:

1. **What**: What is shown
2. **How**: How it was generated
3. **Why**: What to notice

Example:

```markdown
![Phase diagram of spinal stability regimes. The parameter space (K, τ) shows three distinct regions: gravity-dominated (red), cooperative (green), and information-dominated (blue). Generated from numerical solutions of the delayed feedback model with B=1.0 Pa·m².](../figures/phase_diagram.png)
```

### 9. Cross-Reference in Text

Reference figures in your document:

**Markdown:**

```markdown
As shown in the phase diagram (above), three regimes emerge...
```

**LaTeX:**

```latex
As shown in Figure~\ref{fig:phase_diagram}, three regimes emerge...
```

### 10. Version Control Figures

**Git LFS for large files:**

```bash
git lfs track "*.png" "*.pdf"
git add .gitattributes
git add research/figures/
git commit -m "Add phase diagram figure"
```

**Exclude generated figures:**
Add to `.gitignore`:

```
research/figures/temp_*
research/figures/*_draft.*
```

## Best Practices

### File Organization

- Keep figures in dedicated `figures/` directory
- Organize by project/paper if multiple
- Separate raw data, scripts, and final figures

### Naming

- Use descriptive, searchable names
- Include key parameters in name when relevant
- Version important figures explicitly

### Documentation

- Document generation method
- Include script/notebook that created figure
- Note any manual edits or annotations

### Quality

- Use vector formats (PDF, SVG) for plots when possible
- Ensure sufficient resolution (≥300 DPI)
- Use colorblind-friendly palettes
- Include scale bars or axes labels

### Captions

- Make figures self-explanatory
- Define all symbols and abbreviations
- Explain what to focus on

## Troubleshooting

### Figure Not Displaying

**Check path:**

```bash
# Absolute path
ls /Users/mac/life/research/figures/your_figure.png

# Relative path
ls ../figures/your_figure.png
```

**Verify reference:**

```bash
grep "your_figure" research/document.md
```

### Wrong Figure Size

**Markdown (limited control):**

```markdown
![caption](figure.png)  # Uses full width
```

**HTML in Markdown:**

```html
<img src="../figures/figure.png" width="50%" alt="caption">
```

**LaTeX:**

```latex
\includegraphics[width=0.7\textwidth]{figures/figure.pdf}
```

### Figure Quality Issues

**Increase resolution:**

```python
# In matplotlib
plt.figure(dpi=300)
plt.savefig('output.png', dpi=300, bbox_inches='tight')
```

**Anti-aliasing:**

```python
plt.rcParams['text.antialiased'] = True
plt.rcParams['lines.antialiased'] = True
```
