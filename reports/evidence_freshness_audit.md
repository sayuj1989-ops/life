# AFCC Evidence Freshness Audit
**Date:** 2026-02-19
**Status:** Audit Complete

## Executive Summary
An audit of the AFCC run history from 2026-01-06 to present reveals significant redundancy in the daily "refresh" cycles. Multiple consecutive runs produced identical per-gene metrics, suggesting that the "daily refresh" often re-analyzes static data without new inputs or code changes affecting the output.

**Critical Findings:**
1. **Static Intervals:** Several periods show 0 changes across all common genes (e.g., Feb 11-13, Jan 20-21). This implies computational waste or a lack of new candidate ingestion during these windows.
2. **Schema Drift:** A major schema change occurred on Jan 09, standardizing column names (e.g., `anisotropy` -> `anisotropy_index`).
3. **Feb 16 Baseline:** The target baseline for this analysis (2026-02-16) shows 11 changed genes versus the previous run (Feb 13), confirming it represents a distinct and active analysis batch, validating its selection as a reference point.
4. **Post-Baseline Stagnation:** The runs on Feb 18 and `manual_metabolic_update` appear to duplicate previous results (Feb 17 and 18 respectively), cautioning against assuming "newer is better".
